## 1. 기획 의도

### 왜 이 주제인가

한국 청년층에서 약물 관련 사건은 해마다 증가하고 있습니다.  
그러나 "약물 중독"이라는 주제는 여전히 사회적 금기로 취급되어,  
정작 예방 교육이 가장 필요한 10~20대에게는 충분히 닿지 못하고 있습니다.

**LAST BRAKE**는 플레이어가 직접 선택의 주체가 되어  
"왜 사람들이 약물에 빠져드는가"를 체험적으로 이해하도록 설계된 게임입니다.  
도덕적 훈계가 아닌, **반복된 작은 선택이 어떻게 큰 결과를 만드는지**를  
게임이라는 매체로 전달하는 것이 이 기획의 핵심입니다.

### 게임이 아닌 게임

이 작품은 단순한 오락 게임이 아닙니다.  
마지막 장면에서 플레이어는 자신의 선택 결과 수치를 직면하고,  
세 가지 선택지 중 하나로 실제 **정신건강위기상담전화(1393)** 에 연결할 수 있습니다.  
게임을 끝낸 플레이어가 현실의 누군가를 떠올리는 것, 그것이 목표입니다.

## 4. 등장인물

### 주인공

**도윤 (Doyun)**  
평범한 대학생. 클럽 문화에 처음 발을 들이게 되는 인물.  
모든 선택은 도윤의 관점에서 진행되며, 플레이어가 도윤이 된다.  
4가지 감정 표정(Normal · Anxious · Regret · Surprised)이 엔딩마다 다르게 등장해  
주인공의 심리 상태를 시각적으로 전달한다.

### 주변 인물

| 이름 | 관계 | 역할 |
|------|------|------|
| **민재 (Minjae)** | 도윤의 선배 | 약물 세계로 끌어들이는 인물. 처음엔 친근하게 접근 |
| **하준 (Hajun)** | 도윤의 친구 | 위험을 경고하는 존재. 이성적 선택지의 근거가 됨 |
| **서아 (Seoa)** | 도윤을 걱정하는 인물 | 도윤과 가장 가까우며, 노멀 엔딩에서 중요한 역할 |

## 6. 게임플레이 시스템

### 핵심 수치 (3-Stat System)

| 수치 | 색상 | 초기값 | 의미 |
|------|------|--------|------|
| **INT (판단력)** | 초록 | 100 | 높을수록 이성적 선택 가능. ADDICT 60+ 시 특정 선택지 잠금 |
| **RISK (위험도)** | 주황 | 0 | 높을수록 위험한 상황에 노출. 60+ 시 HUD 빨간색 |
| **ADDICT (의존도)** | 보라 | 0 | 높을수록 약물 의존. 60+ 시 강제 선택 발동 |

### 선택지별 수치 변화

**Step 1 — 프롤로그**

| 선택지 | INT | RISK | ADDICT |
|--------|:---:|:----:|:------:|
| 집에 갈게요 | +10 | −5 | 0 |
| 가볼까? (클럽 동행) | −10 | +15 | +5 |

**Step 2 — 클럽**

| 선택지 | INT | RISK | ADDICT |
|--------|:---:|:----:|:------:|
| 거절한다 | +15 | −5 | 0 |
| 한 번만 해본다 | 0 | 0 | +5 |
| 적극적으로 즐긴다 | −20 | +10 | +20 |

**Step 3 — 다음날 아침**

| 선택지 | INT | RISK | ADDICT |
|--------|:---:|:----:|:------:|
| 끊겠다고 결심 | +15 | 0 | −5 |
| 다시 연락한다 | −5 | +5 | +10 |
| 더 구한다 | −10 | +10 | +20 |

**Step 4 — 파티**

| 선택지 | INT | RISK | ADDICT |
|--------|:---:|:----:|:------:|
| 함께한다 | −10 | +10 | +10 |
| 더 강한 걸 요구 | −20 | +15 | +25 |
| 민재 말을 따른다 | −15 | +10 | +30 |

**Step 5 — 무너짐 (엔딩 결정)**

| 선택지 | 조건 | INT | RISK | ADDICT |
|--------|------|:---:|:----:|:------:|
| 경찰에 신고한다 | INT ≥ 70 필요 | +25 | −20 | −20 |
| 다 끝났어... (포기) | 없음 | −15 | +10 | +5 |
| 형 시키는 거 다 할게 | 없음 (강제 배드) | −25 | +20 | +30 |

### 강제 선택 메커닉

**ADDICT ≥ 60** 상태에서 선택지가 등장하면:

1. 이성적 선택지 버튼이 비활성화(빨간색 틴팅)
2. 1.2초 후 나쁜 선택지가 자동으로 클릭되는 애니메이션 재생
3. 의존도에 의해 선택권을 잃어가는 상태를 시각적으로 표현

## 7. 엔딩 분기

### 분기 조건

| 엔딩 | 조건 | 의미 |
|------|------|------|
| **굿 엔딩** 「새벽의 선택」 | INT ≥ 75 AND ADDICT ≤ 40 | 도윤이 스스로 멈출 수 있었던 결말 |
| **노멀 엔딩** 「연락처 삭제」 | INT ≤ 50 AND RISK ≥ 60 | 상처를 안고 일상으로 돌아가는 결말 |
| **배드 엔딩** 「끝나지 않는 밤」 | ADDICT ≥ 80 | 완전히 무너진 결말. 체포와 붕괴 |
| **트루 엔딩** | 세 엔딩 클리어 후 재시작 | 도윤이 직접 플레이어를 바라보는 4번째 벽 |

### 엔딩 시퀀스 — 3단계 메타 구조

모든 엔딩은 동일한 3단계 메타 시퀀스를 거칩니다.

```
[Phase 1 — 메시지]
BGM 페이드 → 도윤 응시 애니메이션 → 화면 흑백화
→ "중독은 특별한 사람이 아니라, 반복된 선택에서 시작됩니다."
→ 암전

[Phase FACE — 얼굴 응시]  ← TrueEnd 전용
화면 가득 도윤 얼굴 → 글리치 + 강한 비네트
→ 표정이 점점 변화 (긴장 고조)
→ 탭/클릭으로 다음 단계

[Phase 2 — 결말 화면]
스탯 수치 공개 애니메이션
→ 엔딩별 도윤 표정
→ "이래도 계속 하겠습니까?"
→ ① 다시 시작  ② 1393 전화  ③ 게임 종료
```

## 8. 시각 효과 시스템

### FX 오버레이 (FXManager)

대화 흐름에 따라 자동 트리거되는 화면 레이어 효과입니다.

| 효과 | 강도 | 용도 |
|------|------|------|
| **RedWarning** | 붉은 플래시 (0.15초 in / 0.45초 out) | 위험한 선택 직후 충격 |
| **Glitch (상시)** | alpha 0.02~0.06, 미세 정적 | ADDICT 상승 시 배경 분위기 |
| **GlitchBurst** | alpha 0.15~0.45, 1.5~2.5초 자동 종료 | 극적 순간 번쩍임 |
| **Vignette** | intensity 0.8 | 긴장감 고조 구간 |
| **BlurPulse** | 3회 반복 페이드 | 정신 혼란·약효 묘사 |
| **BreathingVignette** | 0.25↔0.50, 5초 주기 cos 파형 | 결말 화면 조용한 중독감 |

> 텍스트 가독성 원칙: 대화 중에는 alpha를 0.06 이하로 억제,  
> GlitchBurst는 선택 결과 등 텍스트 없는 순간에만 발동.

### CG 이벤트 (CGViewer)

| 타이밍 | CG | 연출 |
|--------|-----|------|

|------|-----------|
| 스마트폰 | 민재의 문자 수신 |

## 10. UI · UX 설계

### 인게임 HUD (StatHUD)

화면 최상단에 항상 표시되는 3개 수치 바.

```
┌─────────────────────────────────────────────────────────┐
│  이성 [████████░░]  위험도 [████░░░░░░]  의존도 [██░░░░░░░░]  │
└─────────────────────────────────────────────────────────┘
```

- 선택 후 0.6초 애니메이션으로 변화
- RISK / ADDICT 60+ 시 해당 바 빨간색 전환
- Unity Slider 대신 `anchorMax.x` 직접 제어 → 레이아웃 충돌 없음

### 대화 UI (DialogueUI)

- 하단 대화창: 화자명 + 대화 텍스트 타이핑 효과
- 독백 처리: 이탤릭 + 별도 색상
- FX / CG / 소품 컷인을 대화 라인 단위로 트리거

### 결말 화면 (Phase 2) 레이아웃

```
┌──────────────────────────────────────────────────┐
│  [도윤 얼굴]  │  — 게임 결과 —                     │
│               │  판단력   [░░░░░░░░] → 애니메이션  │
│  엔딩별 표정  │  위험도   [░░░░░░░░]               │
│  + 하단 배지  │  의존도   [░░░░░░░░]               │
├──────────────────────────────────────────────────┤
│  도윤: 이래도 계속 하겠습니까?                         │
├──────────────────────────────────────────────────┤
```

### 핵심 기술 포인트

**1. DontDestroyOnLoad 싱글턴 체인**  
FXManager · SFXManager · BGMController · ScreenEffects · StatManager가  
씬 전환 후에도 상태를 유지하며, 11개 씬 전체에서 동일한 인스턴스로 동작합니다.

**2. Runtime Canvas 생성 방식 (Phase 2)**  
결말 화면 전체를 코드로 생성하여 씬 의존성 제거.  
sortingOrder 900 (결말) → 985 (소품) → 998 (FX) → 999 (페이드) 레이어링.

**3. anchorMax.x 수치 바 방식**  
Unity Slider + LayoutGroup 충돌 문제를 해결하기 위해  
Fill 이미지의 anchorMax.x를 직접 lerp하는 방식으로 전면 재작성.

**4. 글리치 이중 모드**  
상시 모드 (alpha 0.02–0.06, 텍스트 가독성 유지)와  
버스트 모드 (alpha 최대 0.45, 자동 종료)를 분리.  
ADDICT 수치에 따라 코드에서 자동 선택.

**5. 호흡 비네트 (cos 파형)**  
```csharp
float ratio = (1f - Mathf.Cos(t / PERIOD * Mathf.PI * 2f)) * 0.5f;
intensity = Mathf.Lerp(MIN_V, MAX_V, ratio);
```
글리치 없이 중독의 무게감을 표현하는 Phase 2 전용 연출.

**6. Reflection 기반 에디터 자동 연결**  
AudioLinker가 `BindingFlags`를 이용해 SFXManager의 37개 AudioClip 필드를  
이름으로 자동 매핑 → 11개 씬 전체를 순회하며 1클릭으로 연결/해제.

### 해결한 주요 버그

| 버그 | 원인 | 해결 |
|------|------|------|
| 결말 화면 뒤로 씬 배경 비침 | 배경 탐색 함수가 60% 투명도로 씬 배경을 사용 | 불투명 단색 배경으로 교체 |
| ADDICT 60+ 시 대화 텍스트 가려짐 | 글리치 alpha 최대 0.45 지속 | 상시 alpha 0.06 이하로 제한, 버스트 분리 |
| Phase 2 대화창 미표시 | NullRef 예외로 코루틴 중단 | null 체크 전면 추가, alpha = 1 즉시 표시 |
| 스탯 바 카드 overflow | 중앙 앵커 기준 계산 오류 | 상단 앵커 기준으로 전환 |
| 씬 전환 후 dangling Instance | DontDestroyOnLoad 후 OnDestroy 미처리 | `if (Instance == this) Instance = null` 추가 |
